## [v0.1.0] - 2026-03-01

### Highlights

- Declarative specification framework for defining complete design systems as machine-readable, LLM-optimized code
- Go SDK with types for 9 canonical layers: Meta, Principles, Foundations, Components, Patterns, Templates, Content, Accessibility, Governance

### Added

- Go SDK (`sdk/go`) as nested module with complete type system ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `Meta` type for system metadata (name, version, maturity, maintainers) ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `Principles` type for design philosophy and brand values ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `Foundations` type for design tokens: colors, typography, spacing, elevation, motion, grid, breakpoints, borders, opacity, z-index ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `Component` type with variants, states, props, slots, and constraints ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `Pattern` type for multi-component UX solutions with layouts ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `Template` type for page-level layouts with regions ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `Content` type for voice, tone, terminology, and microcopy guidelines ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `Accessibility` type for WCAG requirements, keyboard, and screen reader support ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `Governance` type for versioning, contribution, and deprecation policies ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `LLMContext` type for AI code generation optimization (intent, allowed/forbidden contexts, anti-patterns) ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `LoadDesignSystem()` for loading specs from JSON/YAML files or directory structures ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
- `GenerateSchema()` and layer-specific schema generation functions ([`7b0d3a8`](https://github.com/plexusone/design-system-spec/commit/7b0d3a8))
